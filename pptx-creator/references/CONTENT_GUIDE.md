# 内容创作指南

在将源文档转换为 `presentation.json` 时，必须严格遵循以下步骤。内容示例位于 `references/CONTENT_EXAMPLES.md` 文档。

---

## 1. 内容创作工作流 (AI Agent Workflow)

### 第1步：通读并识别叙事弧 (Narrative Arc)

-   **目标**: 理解源文档的整体结构和意图。
-   **操作**:
    1.  通读全文，识别出文章的**开端**（介绍背景、提出问题）、**主体**（核心论点、分析过程、解决方案）和**结尾**（总结、结论、未来展望）。
    2.  根据所选的**听众-目标策略模型**，判断哪些部分是重点，哪些可以省略或简化。例如，为专家做报告时，可以大幅简化“开端”的基础背景部分。

### 第2步：解构为幻灯片序列 (Deconstruct into Slides)

-   **目标**: 将叙事弧的每一部分映射为具体的幻灯片序列。
-   **操作**:
    1.  **开端**: 至少使用一张 `hero-cover` 作为封面。
    2.  **主体**: 每个核心论点或主要步骤都应成为一个 `section-divider`。在其之下，根据**听众-目标策略模型**选择合适的幻灯片类型。例如，`教学`模型多用`pipeline`，`报告`模型多用`chart`。
    3.  **结尾**: 使用 `section-divider` 引入总结，并以 `minimal-quote` 或 `title` (致谢) 幻灯片作为收尾。

### 第3步：为每张幻灯片撰写内容 (Author Each Slide)

-   **目标**: 遵循内容核心原则和所选模型，填充每张幻灯片的数据。
-   **操作**:
    1.  **撰写论点式标题**: 为 `title` 字段撰写一个完整的陈述句（见原则2）。
    2.  **提炼关键词要点**: 根据**策略模型**调整提炼的粒度。`教学`模型可包含更多解释性文字，`报告`模型则应使用高度精炼的术语。
    3.  **优先视觉化**: 始终遵循数据和图片优先的原则（见原则4和5）。

---

## 2. 内容核心原则 (Universal Principles)

必须遵循以下原则。在编写 `presentation.json` 的 `slides` 数组时，逐条对照。

- **一页一重点 (One Idea Per Slide)**: 每张幻灯片只传达一个核心观点。
- **标题即论点 (Title Is Thesis)**: 幻灯片的标题本身应是一个完整的陈述句。
- **要点是关键词 (Bullets are Keywords)**: 正文的要点应是高度精炼的关键词或短语。
- **数据可视化优先 (Data Visualization First)**: 有数字就用图表。
- **图片优先 (Image First)**: 有图就用图文版式。
- **叙事有节奏 (Narrative Pacing)**: 使用分隔页控制节奏，避免观众疲劳。
- **语言干净利落 (Clean Language)**: 使用主动语态，删除填充词。
- **留白即呼吸 (Whitespace is Breathing)**: 严格控制单页总字数和要点数量。

---

### 1. One Idea Per Slide — 一页一个重点

每张幻灯片只传达一个核心观点。如果一个主题有 5 个要点，拆成 2-3 页，而不是挤在同一页。

- ❌ 坏例子：一页讲完"管径确定方法"的 4 个要点 + 流速范围 + 经济流速 + 界限流量表
- ✅ 好例子：一页讲"管径确定方法"，下一页讲"经济流速与界限流量"

### 2. Title Is Thesis — 标题即论点

标题必须是完整的陈述句（至少 3 个字符，至多19个字符），让观众只看标题就理解这页在说什么。不要用名词短语做标题。

- ❌ `"管径确定方法"` → 观众不知道要说什么
- ✅ `"管径由设计流量和经济流速共同确定"` → 一句话说清论点

### 3. Bullets Are Keywords — 要点是关键词，不是完整句子

每条要点 ≤ 40 字，用关键词短语而非完整句子。完整内容由演讲者口述。

- ❌ `"管径按分配后的流量确定，公式为 D = √(4q/πv)"`
- ✅ `"D = √(4q/πv)，由设计流量和流速决定"`

### 4. Data Visualization First — 数据可视化优先

有数字就不要只写文字：

| 数据类型 | 推荐幻灯片类型 |
|:---|:---|
| 关键指标 / 统计数据 | `big-number-grid` |
| 两组对比 | `comparison` |
| 流程 / 步骤 | `pipeline` |
| 图表 / 趋势 | `chart` |
| 图片 / 示意图 | `two-column` 或 `image-grid` |

### 5. Image First — 图片优先

涉及图表、示意图、流程图、照片等视觉元素时，**必须**使用 `two-column` 或 `image-grid`，不得用纯文字 `content`。图片放在右侧（`right.image`），文字要点放在左侧（`left.bullets`）。图片缺失时 builder 自动渲染占位框。

### 6. Narrative Pacing — 叙事节奏

控制 body 类型幻灯片的密度，避免观众疲劳：

- 不超过 4 页连续的 body 类型（`content`、`two-column`、`comparison`、`pipeline`、`big-number-grid`）
- 8+ 页演示文稿必须包含至少 1 个 `hero-cover` 或 `section-divider`
- 结尾用 `minimal-quote` 或 `section-divider` 收束
- 典型节奏：`hero-cover` → `content` × 2 → `section-divider` → `content` × 2 → `minimal-quote`

### 7. Clean Language — 语言干净

- 使用主动语态，不用被动语态
- 删掉填充短语："我们认为"、"我们相信"、"值得一提的是"、"需要注意的是"
- 每条要点只表达一个核心思想
- 幻灯片上用关键词，完整句子由演讲者口述

### 8. Whitespace Is Breathing — 留白是呼吸

- 单页总字数 ≤ 120 字（中文）
- 单页不超过 8 条要点
- `content` 类型至少 3 条要点（少于 3 条建议换用其他布局）
- 不堆满文字，给视觉留空间
